Build model-ready tables for the medical recommendation module.

The main artifact is a patient-condition-medication table: patient_id, condition, medication, label_prescribed, ...

Rows are created by pairing each observed patient condition with the medications that are plausible candidates for that condition, 
based on medications observed for the same indication in the source data. Observed prescriptions are positive labels; unobserved 
condition-specific candidates are negative labels.

Outcome columns are merged as patient-level historical summaries. Treat them as
evaluation/risk context unless you later add temporal cutoffs, because outcomes
can occur after the medication start date.



The model treats each row in patient_condition_medication.csv as a candidate:

patient + condition + diagnosis + medication -> probability medication is prescribed

GNNExplainer aims to identify the **minimal graph structure and node features** responsible for a prediction.

- Identifying important nodes and edges.
- Providing faithful explanations without retraining the model.
- Explaining node, edge, and graph classification tasks.

GNNExplainer explains GNN predictions by learning a compact subgraph and feature mask that maximally preserve the original model prediction.

y^=f(G,X)

where: G = graph, X = node features

Graph Mask            MG           which selects important edges.

Feature Mask          MX           which selects important features.

The explanation graph becomes:

Gs=G⊙MG

where: ⊙  denotes masking.

The key idea is maximizing mutual information:

max⁡GsI(Y,Gs)

where:

- YYY = prediction
- Gs = explanatory subgraph

Find the smallest subgraph that retains the most information about the prediction

### Fidelity

How much prediction quality remains after keeping only explanatory components.

Fidelity=f(Gs)−f(G)

---

### Sparsity

How small the explanation is.

Sparsity=∣Gs∣/∣G∣
